## Contributing

This repository is used for real-world automation workflows. Contributions are welcome, but we keep changes tightly scoped to maintain reliability and security.

### What we accept

- Bug fixes and improved error handling
- Tests and CI improvements
- Better logging/observability
- Safer integrations (rate limits, retries, redaction)

### What to include in a PR

- A clear description of the behavior change and why it matters
- A test plan (ideally automated tests)
- Notes about any security or data-handling implications
